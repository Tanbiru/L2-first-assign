Q2: How do Generics allow you to build reusable components and functions that stay strictly typed regardless of the data structures passed in?

Ans: Generics help a lot when you want to write a function 
that works with types of data. You can still use TypeScripts type checking, which's great. You might have to write the function many times for different types of data like one for numbers and one for words. This is a lot of work.Generics are like a placeholder for a type of data. You can say, I don't know what type of data this will be yet but when you use this function tell me what type of data it is and I will use that type of data everywhere. For example imagine you have a function that takes a list of things and gives you the thing. With generics you can write this function once. It will work for a list of numbers a list of words or even a list of objects. TypeScript will remember what type of data you are using when you call the function. If you give it a list of words TypeScript knows that the first thing will be a word. If you give it a list of numbers TypeScript knows that the first thing will be a number. Generics are really useful for things, like this. They make your code safer and easier to use with generics.
